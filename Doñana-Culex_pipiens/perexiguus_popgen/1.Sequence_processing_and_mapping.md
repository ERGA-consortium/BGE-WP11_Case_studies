# Alignment
Fastqc, fastp and alignment with bwa mem was run on CESGA server (managed with SLURM). The rest of the pipeline were run in poppy server, which has not job management system.
First, I did quality control of raw reads using fastqc and multiqc to decide which filters to apply with fastp. After applying fastp, I re-did fastqc and multiqc to ensure everything looked good.

1. [fastqc.sh](../scripts/fastqc.sh): runs fastqc in all files and then do multiqc over the results. I do it before and after applying fastp to
2.  [list-array-fastp.sh](../scripts/list-array-fastp.sh): it creates a list ready to run fastp as a job array on many files within a directory
3. [fastp-array.sh](../scripts/fastp-array.sh): runs fastp as a job array using all the files contained in a list

Main fastp commands used:
```bash
    --trim_poly_g \
    --trim_poly_x \
    --length_required 30 \
    --correction \
    --detect_adapter_for_pe #\
   --adapter_fasta /mnt/lustre/scratch/nlsas/home/csic/dbl/scc/data/polyG.fasta
``` 
Second round of fastqc was done directly:
```bash
fastqc $LUSTRE/3.repetidas/data/*.fastp.fq.gz -o $LUSTRE/3.repetidas/fastqc
```

I changed the name of chromosomes in the reference genome using _sed_ to make working with them easier
```bash
#an example with mitochondrial genome
sed 's/^>OZ175960\.1.*/>MT/' idCulPerx1.2.fna > idCulPerx1.3.fna
#did the same for 3 chromosomes
```

The final genome is idCulPerx1.1.renamed.fna.
I indexed and created the dictionary
``` bash 
# index for BWA
bwa index idCulPerx1.1.renamed.fna
# general index
samtools faidx idCulPerx1.1.renamed.fna
# dict for gatk (not needed if mapping is done with bwa)
samtools dict idCulPerx1.1.renamed.fna -o idCulPerx1.1.renamed.dict
```

Then i run alignment of the samples against the reference genome using [align-bams-index.sh](../scripts/align-bams-index.sh). (A version with the same parameters but adapted to be used in a server without SLURM)
**BORRAR DE ESE SCRIPT LA PARTE DE INDEX Y SORTED**

Then I used [sorted-rg.sh](../scripts/used_4_perex/sorted-rg.sh) to add information on reading groups and index the bam files using piccard and samtools. 
The script also extract statistics from each bam file using _samtools flagstat_. Those were then summarized using [flagstat-summary.sh](../scripts/used_4_perex/flagstat-summary.sh) to check that mapping was good for all of them.

Then I removed duplicates from all bam files using [mark-duplicates-poppy.sh](../scripts/used_4_perex/mark-duplicates-poppy.sh), which uses *picard MarkDuplicates*.

Finally, as we have some samples with multiple sequencing runs, we merge all runs from same samples using [merge-bams.sh](../scripts/used_4_perex/merge-bams.sh).

# Quality checks of final bams:

1. qualimaps + multibamqc First I did qualimaps of every sample with [qualimap.sh](../scripts/qualimap.sh) 
and then did qualimap multi-bamqc. We added information about the batch of sequencing each sample belonged to in the config file used for multibamqc.

```bash
find /sietch_colab/scamison/talapas/6.perexiguus/qualimaps -mindepth 1 -maxdepth 1 -type d | awk -F/ '{print $NF "\t" $0 "\t3.repetidas"}' > multibamqc_config_file.txt
find /sietch_colab/scamison/talapas/5.perex-primeros-raw/qualimaps -mindepth 1 -maxdepth 1 -type d | awk -F/ '{print $NF "\t" $0 "\t2.mayo"}' >> multibamqc_config_file.txt
mkdir multibamqc
cd  multibamqc
mv ../multibamqc_config_file.txt .
qualimap multi-bamqc -d multibamqc/multibamqc_config_file.txt -outdir "./multibamqc" -outfile "perex_multibamqc_report.pdf" -outformat PDF:H
```

2. check the coverage along the genome We calculated coverage along the genome by using mosdepth
 ```bash
for bam in *.bam; do
echo "checking coverage for $bam"
mosdepth --by 100000 ${bam%.bam} $bam
done
```
Then, we used an R script to plot coverage results [mosdepth-coverage.R](../scripts/mosdepth-coverage.R)
