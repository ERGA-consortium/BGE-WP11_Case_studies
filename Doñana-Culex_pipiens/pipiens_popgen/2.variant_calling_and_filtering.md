### 1.Variant calling 
(vcf env on talapas)
* [snp-calling-bcftools.sh](../scripts/snp-calling-bcftools.sh): This script gets a list of bam files and a list with information about sample's origin population and runs *bcftools mpileup* and *call* for the indicated chromosome. The bam list must contain complete paths.

```bash
#Main parameters used
bcftools mpileup -Ou  --threads $thr -f $reference -q 20 -Q 15 -b $list -r "$chromosome" --annotate "FORMAT/AD,FORMAT/DP,INFO/AD" | \
bcftools call -m  --threads $thr -G $population_file -Ob --format-fields "GQ,GP" -o ${dir}/pipiens123_${chromosome}_pop.bcf
```
### 2. Variant filtering 
(earlgrey - talapas)
(vcf - poppy)

Highly repeated regions/TE  were detected by EarlGrey and filtered out masked in the bcf files using [filter-repeats.sh](../scripts/filter-repeats.sh).
```bash
earlGrey -g idCulPipi1.1.primary.fa -s cxpipiensTest -t 16 -l culex_TE_for_RM.lib -c yes -d yes -o ./ 
```
At this point the bcf files of batch 4 were merged to the rest of the samples, as they were done separetely because they were sequenced much later. I used bcftools merge for that.
```bash
#done for the 3 chrosmosomes separately
bcftools merge $123bcf $4bcf -Ob --threads 30 -o pipiens1234_1_norep.bcf 
```

Then, using [filter-snps-invariants.sh](../scripts/filter-snps-invariants.sh) , we filter out invariants sites and highly repeated regions marked by EarlGrey.

This script also generates reports about missing sites per individual. At this point of the analyses, only individuals that really stood out were excluded. Those samples were the same that previously presented low mapping rates. Another filter based on missing sites per individual will be applied later.

```bash
for bcf in *variant.bcf; do
echo "erasing samples with high missingness $bcf"
name="${bcf%.bcf}"
output="${name}_clean.bcf"
echo "Output file: $output"
bcftools view -s ^$samples $bcf -Ob -o $output
echo "done with $bcf"
done 
```

We generate statistics about depth of coverage and plot the results using [depth_plot.R](../scripts/depth_plot.R) to decide a proper threshold for further filters.
```bash
for bcf in *ldepth; do
name="${bcf%.ldepth}"
number=$(echo "$bcf" | cut -d'_' -f2)
    echo "Running Rscript for $bcf with chrom=$number and base name=$name"
Rscript depth_plot.R \
       "${number}" \
        "${name}.ldepth" \
        "${name}.ldepth.mean" \
       "${name}" 
done
```

Next steps applied with [filter-variants.sh](../scripts/filter-variants.sh):
* retain only biallellic SNPs
* exclude sites with INFO/DP> meanDP*2
* turn to ./. any GT with FORMAT/DP <5
* turn to ./. any GT with FORMAT/GQ <30
* remove sites with >50% missing genotypes

This script also returns the number of variants retained after each step (for FORMAT/DP and FORMAT/GQ the scripts calculate how much sites would be remaining if we applied a <20% missing GT per site filter). It also returns metrics on missing sites per individuals after each step.


Then the bcf for each chromosome were merged into a single bcf file and we plotted missingness per individual ([plot-missingness.R](../scripts/plot-missingness.R)) to decide if any other sample should be removed.

```bash
#TO CANCATENATE THE BCFs
#1) sort bcfs
for bcf in *BDQM5.bcf; do
echo "sorting $bcf"
name="${bcf%.bcf}"
output="${name}_sorted.bcf"
echo "Output file: $output"
bcftools sort --temp-dir /sietch_colab/scamison/talapas/variant_filtering/temp-bcftools -Ob -o $output $bcf
echo "done with $bcf"
done
#2) index bcfs
for bcf in *BDQM5_sorted.bcf; do
echo "indexing $bcf"
bcftools index $bcf
done
#3) concat and index
bcftools concat -Ob --threads 30 -o 530pipiens123_all.bcf 530pipiens1234_1_BDQM5_sorted.bcf 530pipiens1234_2_BDQM5_sorted.bcf 530pipiens1234_3_BDQM5_sorted.bcf 
bcftools index 530pipiens123_all.bcf
#4) recalculate the missingness per individual
vcftools --bcf  530pipiens123_all.bcf --out 530pipiens123_all --missing-indv
bcftools +counts 
```
Then remove individuals with too many missing sites using *bcftools view -s ^$samples*

Then I applied the last filters with [filter_variants_MISyMAF.sh](../scripts/filter_variants_MISyMAF.sh):
* retain snps with <20% missing genotypes
* retain alleles with MAF >= 0.03

After that you will have a bcf file ready for population genomics analyses.

### 3. Second variant filtering
(vcf - poppy)

During population structure analyses (see [3.Population_structure_analysis.md](3.Population_structure_analysis.md)) we wanted to confirm the patterns found were not influenced by missing data or chromosomal inversions (see [4.Chromosomal_inversions.md](4.Chromosomal_inversions.md)). For that, we generated a second dataset with stricter filters, reduced to chromosome 2 because we have not found any inversion in that chromosome.

We first obtained separated vcf for 2q and 2p, the two arms of chromosme 2. The approach was to rank samples according to their missing data and then apply filters removing samples with more missing data until we get a dataset with around 50k snps. We applied this approach to both arms of chromosome 2.

1) Obtain vcf for 2p and 2q separately and calculate the number of snps in each one
```bash
bcftools view -r 2:1-112539609 --threads 12 -Oz -o 2p-53023pipiens1234.vcf.gz 53023pipiens1234_all_clean_BDQMF_filteredGT.vcf.gz ; tabix -p vcf 2p-53023pipiens1234.vcf.gz ; bcftools +counts 2p-53023pipiens1234.vcf.gz
#1.012.320
bcftools view -r 2:113453184-213114244 --threads 12 -Oz -o 2q-53023pipiens1234.vcf.gz 53023pipiens1234_all_clean_BDQMF_filteredGT.vcf.gz ; tabix -p vcf 2q-53023pipiens1234.vcf.gz ; bcftools +counts 2q-53023pipiens1234.vcf.gz
# snps 872.141
```

2)  Calculate missing data per individual for both arms
 ```bash
 vcftools --gzvcf 2q-53023pipiens1234.vcf.gz --missing-indv --out 2q-53023pip1234
 vcftools --gzvcf 2p-53023pipiens1234.vcf.gz --missing-indv --out 2p-53023pip1234
 ```

Then I iterated over samples with more missing data until we get a dataset with around 50k snps. This script generates a txt file (snp_retention_by_removed_samples.txt) with the number of snps retained after removing samples with more missing data (column names: #Num_Samples_Removed    Num_SNPs_Retained). 
Them, I listed the final samples remaining in samples-to-keep-2p.txt and samples-to-keep-2q.txt to apply the filters to the vcf files and get the final datasets for analyses.

```bash
bcftools view -S samples-to-keep-2p.txt -Ou 2p-53023pipiens1234.vcf.gz | \
bcftools view -e 'MAF<0.05 || F_MISSING>0' -Oz -o 2p-pippiens1234-filtered.vcf.gz ; tabix -p vcf 2p-pippiens1234-filtered.vcf.gz

bcftools view -S samples-to-keep-2q.txt -Ou 2q-53023pipiens1234.vcf.gz | \
bcftools view -e 'MAF<0.05 || F_MISSING>0' -Oz -o 2q-pippiens1234-filtered.vcf.gz ; tabix -p vcf 2q-pippiens1234-filtered.vcf.gz
```
In order to be strict, we then did LD prunning, admixture and PCA with the filtered datasets 2q  to check if the patterns found were consistent with the ones obtained with the whole dataset (see [3.Population_structure_analysis.md](3.Population_structure_analysis.md)).
