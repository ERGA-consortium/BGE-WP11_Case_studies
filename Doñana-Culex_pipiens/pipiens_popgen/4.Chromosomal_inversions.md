### 1. Windowed PCA (winpca)

 We used [WinPCA](https://github.com/MoritzBlumer/winpca) to perform windowed PCA analyses.
 We run it for 
```bash 
#windowed PCA with default settings
winpca pca $vcf 3:1-190000000 chr3INVmaf -m 0.05 -i 100000
# make a plot of principal component 1 and color by inversion state
winpca chromplot chr3INVmaf 3:1-190882370 -m metadata.txt -g $color
```
Metadata file contains the information according to which the samples are going to be colored in the plot. 
In our case, _winpca chromplot_ was used as a preliminary visualization and to visually determine aproximate breakpoints positions, because plots are dinamyc and allow large zoom in. The final plot was done with ggplot2 in Rstudio

```r
#window pca plot 
winpca <- read_tsv("data/pip1234_3_wpca.pc_1.tsv.gz")
metadata <- read.table("data/2-metadata-pipiens-217samples.txt", sep="\t", header=T)

# Long format
winpca_long <- winpca %>%
  pivot_longer(cols = -pos, names_to = "individual", values_to = "PC1") %>%
  mutate(pos_mb = pos / 1e6)  # convert to Mb
winpca_long$ID <- winpca_long$individual
winpca_long <- winpca_long %>%
  left_join(metadata, by = "ID")

winpca_long$ecot60 <- factor(            
  winpca_long$ecot60,
  levels = c( "molestus", "pipiens","unassigned")
)

# ---- 2. Plot ---
(win_plot <- ggplot(winpca_long, aes(x = pos_mb, y = PC1, group = individual, color = ecot60)) +

   theme(
     panel.background = element_blank(),      # remove grey background inside plot
     plot.background = element_blank(),       # remove outer white margin
```

### 3. PCA within inversions
Following the identification of breakpoints, we determined regions that felll inside the inversions to explore the grouping of samples based on PCA within those regions. 
We used:
Inversion 1q -- 88Mpb to 100Mpb
Inversion 3p --4.8Mpb to 17.5Mpb
Inversion 3q -- 150Mpb to 160Mpb

Pca was performed using [pca-inversion.sh](../scripts/pca-inversions.sh) and the results were plotted in Rstudio using ggplot2. This PCA allowed to divide samples in three groups for each inversion (putative karyotypes). ### 4. FST metrics
We calculated Fst estimates for each chromosome and inversion using [pixy_fst_inversions.sh](../scripts/pixy_fst_inversions.sh). This script uses pixy to calculate Fst estimates for each chromosome comparing samples with different karyotypes (based on PCA results)
The analyses was run 3 times, 1 for inversion. The popmap each time was generated based on the PCA results for the specific inversion. In short, we run the script one time for chromosome 1 (popmap based on 1q) and two times for chromosome 3 (popmap based on 3p and popmap based on 3q). The results were plotted in Rstudio using ggplot2.
